Парадигми:

Различните езици за програмиране поддържат различни стилове на програмиране (парадигми). Избирането на език е предмет на много съображения, като политиката на компанията, пригодността му за дадена цел, наличност на библиотеки и т.н. или лично предпочитание. В най-добрия случай езикът най-добре пригоден за задачата ще бъде избран. В такъв случай остава само да се направи отбор от програмисти, които знаят езика.

Императивното програмиране (от латински: imperare = нареждам, заповядвам) е парадигма в програмирането, спрямо която една програма се състои от редица от команди, които определят какво и в какъв ред компютърът трябва да направи. Това е е най-старата парадигма в програмирането. Примери за програмни езици, ползващи тази парадигма са ALGOL, Fortran, Pascal, ADA, PL/I, Cobol, C и асемблерните езици.

Функционалното програмиране е парадигма, която третира като изчислява оценката на математически функции и избягва променящите състоянието непостоянни данни. Това е декларативна парадигма за програмиране, което означава, че програмирането се извършва с изрази. Във функционален код, изходната стойност на функция зависи само от аргументите, които са вложени по време на функцията, така призованата функция е два пъти с една и съща стойност за един аргумент Х и тя ще произведе същия резултат (Х) всеки път. Премахване на нежелани реакции, т.е. промени в състоянието, което не зависи от входовете на функцията, може да направи много по-лесно разбирането и да се предскаже поведението на една програма , което е един от основните мотиви за развитието на функционално програмиране.

Логическото програмиране е парадигма за програмиране, която се основава на принципите на формалната логика и предикатното смятане. Програмите, написани на език за логическо програмиране, представляват поредица от правила и факти.
Правилата са написани във формата на клаузи: 

H :- B1, ..., Bn.

Правилото има логически смисъл на импликация, която се чете „H е вярно, ако е вярно B1, ..., Bn“: 

H if B1 and ... and Bn.

Правилата имат глава (следствие) H и тяло (опашка, предпоставки) B1, ..., Bn. Правилата дават информация за условията, които са достатъчни за изпълняването на дадена връзка или свойство. Главата на правилото се състои от единствен атом и означава свойството или връзката, която ще е в сила, ако са изпълнени всички предпоставки на правилото. Тялото на правилото се състои от поредица от атоми. Знакът, разделящ тялото и главата, се чете „е вярно, ако“. Основни езици за логическо програмиране са Prolog, ASP и Datalog.

Обектно-ориентираното програмиране (ООП) е парадигма в компютърното програмиране, при която една програмна система се моделира като набор от обекти, които взаимодействат помежду си, за разлика от традиционното виждане, в което една програма е списък от инструкции, които компютърът изпълнява. Всеки обект е способен да получава съобщения, обработва данни и праща съобщения на други обекти. Твърди се, че обектно-ориентираното програмиране дава повече гъвкавост, като прави по-лесно променянето на програми. То намира широка популярност в софтуерното инженерство на мащабни проекти. Поддръжниците на ООП заявяват, че ООП е по-лесно за учене от начинаещи програмисти, за разлика от по-ранни подходи и методики, както и че ООП подходът е по-прост за разработване и поддържане.
Пример за моделиране на форми чрез ООП в Java: 


public abstract class Shape {
    public abstract double area();
    public abstract double perimeter();
}

public class Rectangle extends Shape {
    private double width, length; // страни

    public Rectangle(double width, double length) {
        this.width = width;
        this.length = length;
    }

    @Override
    public double area() {
        return width * length; // лице
    }

    @Override
    public double perimeter() {
        return 2 * (width + length); // периметър
    }
}

public class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }

    public double perimeter() {
        return 2 * Math.PI * radius;
    }
}

public class Triangle extends Shape {
    private double a, b, c; 

    public Triangle(double a, double b, double c) {
        this.a = a;
        this.b = b;
        this.c = c;
    }

    @Override
    public double area() {
        // Херонова формула:
        double s = (a + b + c) / 2;
        return Math.sqrt(s * (s - a) * (s - b) * (s - c));
    }

    @Override
    public double perimeter() {
        return a + b + c;
    }
}

Ниво на езици:

Езиците за програмиране образуват спектър от "ниско ниво" до "високо ниво"; езиците от ниско ниво са по-често машинно ориентирани и вървят по-бързо, докато езиците от високо ниво са по-абстрактни и се работи по-лесно с тях, но програми писани на тах се изпълняват по-бавно. Те позволяват на програмистите да посочват изчисления, въвеждайки формула директно (пр., Y = X^2 + 5*X + 9). FORTRAN, първият широко разпространен език от високо ниво излиза през 1957 г. и много други езици последват: COBOL, който е ориентиран към обработване на комерсиални данни, и Lisp - за изследователски цели.

Обикновенно се програмира по-лесно на езици от високо ниво, отколкото на езици от ниско ниво.

Детайлите изглеждат различно във всеки език, но няколко основни способностти се откриват в почти всеки език:

  • Входни данни от клавиатура, файл или друго устройство
  • Изходни данни на екрана или изпращане на данни на друго устройство
  • Аритметични операции като събиране и умножение.
  • Условно изпъление: проверка на някакви условия и изпълнение на съответни конкретни инструкции
  • Повтаряне на някакво действие многократно, обикновенно с някаква вариация.
